---
name: No-streak doctrine for Soft Landing
description: Hard prohibition on streak counters, percentages, charts, and guilt-driven notifications in any retention surface
type: feedback
---

Soft Landing must never ship: streak counters, "X days in a row" copy, fire/flame icons, percentage stats ("67% of check-ins were tired"), bar charts, pie charts, calendar heatmaps, or guilt-pattern push notifications ("We miss you" / "Don't break your streak").

**Why:** Soft Landing's audience is faith-based wellness users, not productivity-app users. Conventional habit-loop dopamine creates anxiety-on-absence — a user feels guilty for missing a day. This is poison for this audience and breaks the brand's core promise of being a soft place to land. The competitive moat IS the discipline to refuse these patterns; every other wellness app will add streaks because PMs measure DAU. Restraint is the differentiator. The app must feel like opening a paper journal, not maintaining a metric.

**How to apply:** Any spec, review, or implementation that introduces a numeric retention metric in user-facing UI must be flagged 🔴 Critical. Frequency awareness should always be surfaced as natural-language sentences ("Tired has visited most this week") and tally-mark dots in deep views, never as raw counts in primary surfaces. The Pattern tab is a *letter*, never a dashboard. Notifications must be invitational ("How was today?" / "There's a verse waiting, if you'd like one"), never guilt-inducing, never with exclamation marks, never with "Open the app" imperatives.
