name: Pattern / Held / Return three-pillar system
description: Foundational vocabulary and design rules for Soft Landing's reflective retention architecture (frequency, memory, weekly observation letter)
type: project
---

The Soft Landing emotional-tracking and retention architecture rests on three named primitives. Use these names verbatim in all design work — they are the load-bearing vocabulary.

**The Held** — the user's collection of saved verses + generated letters. Replaces "History" tab. Mental model: a kept book, not a feed. Grouped By Feeling primarily, By Time secondarily. Letter is geographically *beneath* the verse on the same card (wax seal expands card downward), never on a separate screen.

**The Pattern** — weekly observation letter from the app to the user, written in Lora 18px in the same envelope/seal aesthetic as validating messages. Generated by Claude with an "observation, not validation" system prompt. Lives in a fourth nav tab `❋ Pattern`, gated until 7th check-in. Archive of past Pattern letters becomes the longitudinal emotional autobiography moat. NEVER a dashboard, chart, heatmap, or percentage. Always a letter, max 180 words, signed with a single warm word (Held. / Witnessed. / Beside you. / Quietly. / Yours.).

**The Return** — when user checks in under an emotion they've sat with before, app sometimes (~30% probability when eligible) surfaces a previously-saved verse with copy "Last time you were tired, this verse held you." Choice presented: "Sit with this again" or "Receive something new." This is the variable-reward retention engine.

**Why:** The user requested a deep psychological + UX analysis for retention hooks that feel warm not gamified, plus a verse/letter/emotion combo showcase, plus a non-clinical pattern view. This three-pillar vocabulary emerged as the cleanest naming and was committed to in the spec returned to the user on 2026-05-02.

**How to apply:** Any future spec touching history, saved verses, letter relationships, emotional tracking, or retention mechanics must use Held / Pattern / Return as the canonical primitive names. Frequency must always surface as natural language sentences ("This one keeps finding you" / "You've held this one before"), never numbers in the primary view. Tally-mark dots (• • •) for save counts in detailed views. Letters live geographically beneath verses, never beside or after.
